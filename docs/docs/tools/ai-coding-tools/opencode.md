### Key Differences from Antigravity / Gemini CLI
- **No background Sidecars** — OpenCode does not manage background daemons or cron schedules
- **No lifecycle Hooks** — OpenCode does not intercept terminal or file actions through pre/post-hook event pipelines
- **No packaged Plugins** — OpenCode does not package extensions into namespaced plugin folders
- Extensibility is handled via **MCP Servers**, **LSP Server configurations**, and **Markdown-based custom commands / skills / rules**

### Naming Convention
- **Skill naming**: snake_case
- **Agent naming**: snake_case
- Commands and skills are in **separate directories** (unlike Claude Code where commands are skills with `disable-model-invocation: true`)
- **Project memory**: `AGENTS.md` at workspace root (generated by `/init`)
- **Config location**: `~/.config/opencode/`

## Built-in Tools

| Tool | Permission Setting |
|------|-------------------|
| `bash` — Execute shell commands | `bash` |
| `edit` — Modify files via exact string replacement | `edit` (covers edit/write/apply_patch) |
| `write` — Create new files or overwrite existing | same as `edit` |
| `read` — Read file contents (supports line ranges) | `read` |
| `grep` — Search file contents with regex (ripgrep-based) | `grep` |
| `glob` — Find files by pattern matching | `glob` |
| `lsp` (experimental) — Code intelligence via LSP (requires env var) | `lsp` |
| `apply_patch` — Apply patch/diff files | same as `edit` |
| `skill` — Load a SKILL.md skill file | `skill` |
| `todowrite` — Manage todo lists during complex operations | `todowrite` |
| `webfetch` — Fetch and read web pages | `webfetch` |
| `websearch` — Web search via Exa AI (requires env var or OpenCode provider) | `websearch` |
| `question` — Ask user questions with multiple-choice options | `question` |

All tool names are **lowercase** (unique among AI coding tools).

## Configuration

- **Server config**: `~/.config/opencode/opencode.json` (or `.opencode/opencode.json`)
- **TUI config**: `~/.config/opencode/tui.json` (separate)
- **Format**: JSON / JSONC
- **Schema URL**: `https://opencode.ai/config.json`
- **Merge**: Deep merge (project overrides global)

### Config Sections
- `permission` — Tool-level allow/deny/ask rules
- `model` — Default model provider
- `customTools` — Define custom tools
- `mcpServers` — MCP server configuration
- `language` — Language setting
- `policies` — Enterprise policies

### Permission Rules
Permission values per tool:
- `"allow"` — No prompt, always allowed
- `"deny"` — Blocked
- `"ask"` — Ask user each time (default)

### MCP Configuration (Harness)
```json
{
  "mcp": {
    "kubernetes": {
      "type": "local",
      "command": ["npx", "-y", "kubernetes-mcp-server@latest"],
      "enabled": true
    },
    "aspire-dashboard": {
      "type": "remote",
      "url": "http://localhost:18891",
      "enabled": false
    }
  }
}
```
Disable syntax: `"enabled": false`

## Agents

**Primary agents** (switchable via Tab key):
- **Build** — Default, all tools enabled, full development access
- **Plan** — Read-only, analysis + planning, denied file edits by default

**Subagents** (invoke via `@agentname`):
- **General** — Multi-step tasks, full tool access (except todo)
- **Explore** — Fast read-only codebase exploration
- **Scout** — External docs/dependency research (clones repos to managed cache)

## Custom Tools

Define custom tools in `opencode.json`:
```json
{
  "customTools": {
    "my-tool": {
      "command": "npx",
      "args": ["my-package"]
    }
  }
}
```

## MCP Support

Configured in `opencode.json` → `mcpServers` section:
```json
{
  "mcpServers": {
    "server-name": {
      "command": "npx",
      "args": ["-y", "package-name"],
      "enabled": true
    }
  }
}
```
- Supports STDIO and HTTP/SSE transport
- Per-server enable/disable via `"enabled": false`
- All MCP tools can be controlled via `permission` rules

## Skills System

- Format: `SKILL.md` with YAML frontmatter
- Naming: snake_case directory names
- Tool names: lowercase
- Frontmatter: name, description, allowed-tools, docs-url, disable-model-invocation, on (trigger keywords)
- Activation: auto on matching keywords (`on` field), or manual via `/name`
- Keywords: `on` array field triggers skill on matching user input

## Commands (Slash)

- Location: `commands/` directory
- Format: Markdown files
- Template: Go templates for arguments
- Custom commands supported

**Built-in slash commands:**
`/help`, `/init`, `/connect`, `/model`, `/session`, `/add-dir`, `/undo`, `/redo`, `/share`, `/edit`, `/open`, `/config`, `/compact`, `/clear`, `/exit`
