## Approvals / Sandbox

4 approval modes:
- **Auto**: No prompts, run everything
- **Manual**: User must approve each action
- **On-request**: Approve only when asked
- **On-error**: Approve only on errors

Sandbox is OS-enforced. Configurable via `config.toml` or `--ask-for-approval` flag.

## Configuration

- **File**: `~/.codex/config.toml` (TOML format â€” unique among tools)
- **Global only**: No project-level config (unlike other tools)
- **CLI overrides**: `-c key=value` flags

## Skills Format (Dual File)

Codex uses a **unique dual-file format**:

### `skill.yaml`
```yaml
name: my-skill
description: Skill description
model: gpt-4o           # Optional: model override
tools: [read, edit]     # Optional: tool restrictions
version: 1
temperature: 0.3        # Optional: temperature override
```

### `prompt.md`
```markdown
# Skill instructions
The main content of the skill.
```

Optional `setup.sh` script in the same directory for setup automation.

This is the **only tool** that uses a dual-file format + optional setup script.
