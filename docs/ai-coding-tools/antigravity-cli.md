## ⚠️ Gemini CLI Deprecation

**Gemini CLI is being deprecated** as of May 19, 2026. It stops serving requests **June 18, 2026**.

What this means:
- All Gemini CLI config (`~/.gemini/`) and skills must be **migrated to Antigravity**
- Antigravity CLI shares the agent harness with Antigravity 2.0 (desktop GUI)
- Built in **Go** (not Node.js) — faster startup than Gemini CLI
- Key new features: async background workflows, plugins, session export to GUI

### Migration Path
- Skills: Gemini `SKILL.md` → Antigravity `SKILL.md` (compatible format)
- MCP servers: `~/.gemini/config/config.json` → `~/.antigravity/settings.json`
- Commands: TOML format in `commands/` (shared between both)
- Hooks/Sidecars/Plugins: Same format, new config path

## Subagent System (Async)

Antigravity's subagent system is **asynchronous** — subagents run in the background while the parent continues working.

### Invoking Subagents
- Parent calls `invoke_subagent` tool to spawn a concurrent session
- **Workspace options**: inherit parent workspace or create isolated Git worktree
- **Context isolation**: subagent starts with clean context, does not inherit parent history
- **Monitoring**: track progress via subagent panel

### Lifecycle States
| State | Description |
|-------|-------------|
| **Running** | Actively executing tasks. Can be cancelled via Stop button |
| **Idle** | Completed task, awaiting messages. Can be re-awakened |
| **Killed** | Permanently terminated. Worktrees cleaned up. Transcripts still viewable |

### Inter-Agent Communication
- Agents communicate via unique conversation IDs
- Flexible routing: any agent can message any other known agent
- Auto-wake: idle agents wake on receiving a message
- Shared transcripts: agents can view each other's conversation histories

### Built-In Subagents
- **`research`**: Optimized for codebase research and exploration
- **`browser`**: Sandboxed web browser for interactive tasks (via `/browser`)
- **`self`**: Clone of calling agent, identical system prompt and toolsets

### Custom Subagents
- Use `define_subagent` tool to create custom system prompts and tool sets
- Once defined, can be invoked repeatedly in the session
- Max nesting depth: **10 levels**

### Permissions
- Subagents inherit parent's allowed command prefixes and file read/write scopes
- Permission requests bubble up to parent subagent panel UI

### Multi-Agent Teamwork (Ultra Plan)
- `/teamwork-preview` command (Ultra $200/mo plan)
- Collaborative multi-agent framework with error recovery and automatic retries

## Hooks System

Hooks are interceptors at specific lifecycle stages. Configured via `hooks.json`.

### Locations
- **Global**: `~/.gemini/config/hooks.json` or `~/.gemini/config/plugins/<name>/hooks.json`
- **Workspace**: `<workspace-root>/.agents/hooks.json`

### Lifecycle Events
| Event | When It Fires |
|-------|---------------|
| **PreInvocation** | Before model is prompted/invoked |
| **PreToolUse** | Immediately before tool execution |
| **PostToolUse** | Immediately after tool execution |
| **PostInvocation** | After model call completes |
| **Stop** | When execution loop terminates |

### Hook Types (SDK Model)
| Type | Nature | Mode | Use Case |
|------|--------|------|----------|
| **Inspect** | Read-Only | Non-blocking | Tracing, telemetry, auditing |
| **Decide** | Read-Only | Blocking | Access control, guardrails |
| **Transform** | Modifying | Blocking | Data sanitization, prompt augmentation |

### `hooks.json` Example
```json
{
  "hooks": [
    {
      "event": "PreToolUse",
      "tool": "run_command",
      "command": "python3 hooks/verify_command_safety.py",
      "type": "Decide"
    },
    {
      "event": "PostToolUse",
      "tool": "write_file",
      "command": "eslint --fix ${filePath}",
      "type": "Transform"
    }
  ]
}
```

## Sidecars

Background processes managed by Antigravity. Lifecycle includes auto-launch and restart.

### Configuration
- **Global**: `~/.gemini/config/sidecars/<sidecarName>/sidecar.json`
- **Plugin**: `~/.gemini/config/plugins/<pluginName>/sidecars/<name>/sidecar.json`

### `sidecar.json` Schema
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `command` | string | Yes* | Executable to run |
| `builtin` | string | Yes* | Built-in command (e.g. `schedule`) |
| `args` | string[] | No | Arguments to command/builtin |
| `restart_policy` | string | No | `always` (default), `on-failure`, `never` |
| `description` | string | No | Human-readable description |
| `env` | object | No | Environment variables |
| `display_name` | string | No | UI display name |

*Mutually exclusive: one of `command` or `builtin` required.

### Built-in Scheduler
```json
{
  "builtin": "schedule",
  "args": ["0 * * * *", "agentapi", "new-conversation", "Triage review requests."]
}
```

### Runtime Data
Stored at `~/.gemini/antigravity/sidecar_data/<sidecarId>/`:
- `data/` — Persistent data (`ANTIGRAVITY_EXECUTABLE_DATA_DIR`)
- `logs/` — Timestamped stdout/stderr logs
- `events/` — JSON files for `agentapi` calls

### `agentapi` CLI
- `agentapi new-conversation <prompt>` — Create conversation (requires `projectId`)
- `agentapi send-message <conversation_id> <prompt>` — Send to existing conversation

## Plugin System

Plugins are namespaced bundles. A `plugin.json` manifest marks a directory as a plugin.

### Plugin Structure
```
plugins/<plugin-name>/
├── plugin.json       # Required: {"name": "my-custom-plugin"}
├── mcp_config.json   # Optional MCP server definitions
├── hooks.json        # Optional hooks definition
├── skills/           # Optional: skills/<name>/SKILL.md
└── rules/            # Optional: rules/<rule-name>.md
```

### Supported Components
- **Skills**: `skills/<name>/SKILL.md` — instruction files
- **Rules**: `rules/<name>.md` — constraints and guidelines
- **MCP Servers**: `mcp_config.json` — external tool connections
- **Hooks**: `hooks.json` — lifecycle event triggers

### Installation
- **Google-bundled**: Browse from Customizations page
- **Manual workspace**: `.agents/plugins/` or `_agents/plugins/`
- **Manual global**: `~/.gemini/config/plugins/`
