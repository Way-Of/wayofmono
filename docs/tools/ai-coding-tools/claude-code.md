## In This Monorepo

### Harness Integration
The AI Engineering Harness at `packages/@aiengineeringharness/claude/` deploys to `~/.claude/`:

| Component | Location | Description |
|-----------|----------|-------------|
| **Agents** | `agents/` | 6 agents (snake_case): codebase_analyzer, codebase_locator, codebase_pattern_finder, thoughts_analyzer, thoughts_locator, web_search_researcher |
| **Skills** | `skills/` | 35+ skills (13 manual + 22+ auto-triggered), SKILL.md format |
| **Settings** | `settings.json` | Model: `sonnet`, enabled plugin: `gopls-lsp@claude-plugins-official` |
| **MCP** | `.mcp.json` | MCP servers: kubernetes (disabled), aspire-dashboard (disabled, HTTP) |

### Configuration Hierarchy
1. **Managed**: Highest precedence, server-level/IT-enforced settings
2. **User (Global)**: `~/.claude/` — affects all projects
3. **Project**: `.claude/` or workspace root — shared via Git
4. **Local**: `.claude/settings.local.json` — personal overrides (gitignored)

### Project Steering
- **`CLAUDE.md`**: Primary instruction manual, auto-read at every session start. Generated by `/init`
- **`.claude/rules/`**: Modular rule Markdown files with YAML frontmatter and path pattern targeting
- **`@` Referencing**: Reference files in conversation (e.g. `@docs/style-guide.md`)

### Key Differences from Antigravity / Gemini CLI
- **No built-in Sidecars** — Claude Code does not manage background sidecars or cron schedules
- **No Pre/Post Hooks** — Claude Code does not support intercepting commands or tool actions with pre/post-hook scripts
- Commands are implemented as **skills with `disable-model-invocation: true`**

### Naming Convention
- **Skill naming**: snake_case
- **Agent naming**: snake_case
- **Tool names in allowed-tools**: PascalCase (`Read`, `Write`, `Edit`, `Bash`, `Grep`, `Glob`)
- **Project memory**: `CLAUDE.md`
- **Config location**: `~/.claude/`

## Configuration

- **Global**: `~/.claude/settings.json` (JSON)
- **Project**: `.claude/settings.json` (overrides global)
- **MCP**: `.claude/.mcp.json` or embedded in `settings.json` → `mcpServers`
- **Project memory**: `CLAUDE.md` at project root
- **Rules**: `.claude/rules/` directory for modular instructions (.md files with YAML frontmatter)
- **Auto memory**: Claude saves learnings across sessions automatically

### Permission Rules

Allow/deny patterns: `ToolName(specifier)` format:
- `Bash(npm run *)` — Command pattern matching
- `Read(~/secrets/**)` — Path pattern matching
- `Edit(/src/**)` — Path pattern matching for edit/write
- `WebFetch(domain:example.com)` — Domain matching

### CLI Flags
- `--allowedTools` / `--allowed-tools` — Tools that skip permission prompt
- `--disallowedTools` / `--disallowed-tools` — Deny rules
- `--tools` — Restrict which built-in tools are available
- `--model` — Set model (sonnet/opus/haiku/fable or full model ID)
- `--bare` — Minimal mode (skip auto-discovery of hooks/skills/plugins/MCP)
- `--mcp-config` — Load MCP servers from JSON files
- `--advisor` — Set advisor model

### Harness `settings.json`
```json
{
  "$schema": "https://json.schemastore.org/claude-code-settings.json",
  "model": "sonnet",
  "enabledPlugins": {
    "gopls-lsp@claude-plugins-official": true
  }
}
```

### Harness `.mcp.json`
```json
{
  "mcpServers": {
    "kubernetes": {
      "command": "npx", "args": ["-y", "kubernetes-mcp-server@latest"],
      "disabled": true
    },
    "aspire-dashboard": {
      "type": "http", "url": "http://localhost:18891",
      "disabled": true
    }
  }
}
```
Disable syntax: `"disabled": true`

## Skills System

### Storage Locations
1. **Global**: `~/.claude/skills/<skill-folder>/`
2. **Workspace**: `<workspace-root>/.claude/skills/<skill-folder>/`

### SKILL.md Schema
```yaml
---
name: my_custom_skill
description: Performs refactoring of typescript code to adhere to clean-code standards.
allowed-tools: read grep find
disable-model-invocation: true  # Makes it a manual slash command
---
```

### Supported Frontmatter Fields
- **`name`**: Unique name matching folder name
- **`description`**: Summary explaining when agent should activate
- **`allowed-tools`**: (Optional) Space-separated tool list
- **`disable-model-invocation`**: (Optional) When `true`, requires explicit `/skill:name` or `/name` call

### Execution Flow
1. Agent parses available skill descriptions on user request
2. Reads matching `SKILL.md` completely
3. Executes strictly following defined conventions and steps

## Subagent System

- **Agent tool**: Spawns subagents with own context window
- **Foreground**: Shows permission prompts in real-time
- **Background**: Auto-runs with granted permissions, denies on unknown
- **Fork mode**: Inherits full parent conversation
- **Workflow tool**: Orchestrates many subagents, returns consolidated result
- **maxTurns**: Configurable cap on subagent iterations

## Agent Teams (Experimental)

Requires `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`:
- **TeamCreate** — Creates team with multiple teammates
- **TeamDelete** — Disbands team
- **SendMessage** — Inter-agent messaging

## Scheduled Tasks

- **CronCreate/Delete/List**: Session-scoped recurring or one-shot prompts
- **ScheduleWakeup**: Self-paced `/loop` interval
- **RemoteTrigger**: Routines on claude.ai (Pro/Max/Team/Enterprise)

## Cloud Sessions

- **Cloud**: Anthropic-managed VMs for offloaded tasks
- **Remote Control**: Browser-based control of local machine
- **Background agents**: Parallel sessions from one screen

## Additional Features

- **Worktrees**: Isolated git worktrees for safe experimentation
- **Monitor tool**: Background process watching (tail logs, poll CI, watch files)
- **PDF reading**: Short PDFs read whole, long PDFs read in ranges
- **Image support**: Read tool handles PNG/JPG as visual content
- **PowerShell tool**: Native PowerShell on Windows (opt-in on Linux/Mac)
