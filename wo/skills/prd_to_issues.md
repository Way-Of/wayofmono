# PRD to Issues

This skill transforms a high-level Product Requirements Document (PRD) into a backlog of independent tasks.

## Strategy
- **Vertical Slices:** Each issue should deliver a demoable piece of functionality.
- **Dependency Mapping:** Clearly define which issues must be finished before others can start.
- **Granularity:** Each issue should be small enough to be implemented and reviewed in a single cycle.
